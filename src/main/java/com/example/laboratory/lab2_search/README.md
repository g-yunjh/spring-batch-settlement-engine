### [LAB-02] 조회 지연 시간 실험 결과 (10만 건)

| 조회 방식 | 지연 시간 (ms) | 비고 |
| --- | --- | --- |
| **Full Scan (DB)** | **19.43 ms** | 인덱스 없이 전체 탐색 (가장 느림) |
| **Index Scan (DB)** | **17.28 ms** | 인덱스 활용 (데이터 양이 적어 효과 미비) |
| **Redis Cache** | **0.99 ms** | 메모리 기반 조회 (**DB 대비 약 20배 빠름**) |

### 핵심 요약

1. **메모리의 압도적 승리**: Redis()는 DB()보다 **20배** 빠릅니다. 물리적 디스크 I/O 여부가 성능을 결정합니다.
2. **인덱스 효율 한계**: 10만 건 규모에서는 Full Scan과 Index Scan의 차이가 **1.1배**에 불과합니다. (데이터가 백만 건 단위로 커져야 인덱스의 진가가 드러납니다.)
3. **결론**: 초저지연(Low Latency)이 필수인 환경에서는 DB 인덱스 튜닝보다 **Redis 캐시 도입**이 가장 확실한 해결책입니다.
